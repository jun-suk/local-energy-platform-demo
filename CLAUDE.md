# 로컬에너지플랫폼 프로덕트 개발

## 회사 & 프로젝트 컨텍스트

**회사**: 루트에너지 — 지역주민이 지역 재생에너지 사업에 직접 투자할 수 있도록 지원하는 온라인투자연계금융업(P2P) 플랫폼

**프로젝트 목표**: 로컬에너지 투자 플랫폼 프로토타입 개발
- 기획자(준석)가 Claude Code로 바이브 코딩해서 화면/기능 프로토타입 제작
- 백엔드 개발자가 DB 설계 및 API 개발에 참고할 수 있는 명세 제공
- 최종 산출물은 파일로 직접 공유 (HTML 파일 기준)

## 기술 스택

- **프론트엔드**: 순수 HTML / CSS / Vanilla JS (빌드 도구 없음 — 파일을 바로 브라우저에서 열 수 있어야 함)
- **스타일**: CSS Variables 활용, 반응형 고려
- **백엔드**: 미정 (DB 스키마 및 API 명세는 `/docs` 폴더에 별도 문서로 관리)
- **협업**: 파일 직접 공유 + GitHub (보조)

## 폴더 구조

```
02_개발/
├── CLAUDE.md              ← 이 파일 (Claude Code 컨텍스트)
├── index.html             ← 메인 진입점 / 페이지 목록
├── pages/                 ← 화면별 HTML 파일
├── components/            ← 재사용 컴포넌트 (헤더, 카드 등)
├── assets/
│   ├── css/               ← 공통 스타일
│   ├── js/                ← 공통 스크립트 / 유틸
│   └── images/            ← 이미지 리소스
└── docs/                  ← 백엔드 개발자 참고용 명세
    ├── db-schema.md        ← DB 테이블 설계
    ├── api-spec.md         ← API 엔드포인트 명세
    └── data-flow.md        ← 주요 데이터 흐름
```

## 개발 원칙

1. **파일 독립성**: 각 HTML 파일은 브라우저에서 바로 열 수 있어야 함 (서버 불필요)
2. **공통 스타일**: `assets/css/global.css` 를 모든 페이지에서 import
3. **공통 JS**: `assets/js/utils.js` 에 공통 함수 모음
4. **더미 데이터**: API 연동 전에는 JS 내 하드코딩 더미 데이터 사용
5. **백엔드 명세 병행**: 새 화면/기능 만들 때 `docs/` 폴더에 관련 DB/API 명세도 함께 업데이트

## 주요 도메인 개념

- **투자 상품**: 재생에너지 사업(태양광, 풍력 등)에 대한 P2P 투자 상품
- **투자자**: 해당 지역 주민 또는 일반 투자자
- **사업체**: 재생에너지 발전 사업자 (차입자)
- **펀딩**: 투자 상품 모집 → 목표금액 달성 → 실행 → 상환 흐름
- **수익 분배**: 발전 수익 기반 이자 지급

## 작업 방식

- 새 화면 요청 시 → `pages/[화면명].html` 로 생성
- 공통 요소 분리 필요 시 → `components/[컴포넌트명].html`
- DB 테이블 추가 시 → `docs/db-schema.md` 에 append
- API 추가 시 → `docs/api-spec.md` 에 append

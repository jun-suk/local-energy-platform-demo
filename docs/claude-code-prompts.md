## 작업 2 — 백엔드 핸드오프 버전 (`backend-handoff.html`)

```
design-main.html, app.js, styles.css, docs/기능명세서_v3.md 를 읽어줘.
(기능명세서_v3.md 없으면 아래 내용으로 대체)

backend-handoff.html 을 만들어줘.
백엔드 개발자에게 기획 의도를 전달하고, 개발 착수 전 전체 화면을 보여주는 단일 파일이야.

[포함할 화면 — app.js 기존 뷰 그대로]
- home (홈 · 동네소식 피드)
- chat-free (무엇이든 물어보세요 — AI 자유문의)
- dashboard (지역 대시보드 · 지도)
- coop (협동조합 현황)
- coop-members (조합원 명부)
- coop-vote (총회 · 전자투표)
- admin-home (운영 대시보드)
- admin-docs (문서 관리)
- admin-logs (질의 로그)
- admin-coop (조합 운영 · 배당 파이프라인)
- [신규] coop-onboard (조합원 온보딩) ← 아래 상세 설명

[제외할 화면]
- chat-haebit (햇빛소득 진단) — 외주 별도 개발 예정, 이 버전에서 완전 제외

[신규 화면: coop-onboard — 조합원 온보딩 플로우]
사이드바에 "조합원 등록" 메뉴 추가. 아래 4단계 탭/스텝 UI로 구성.

Step 1 · 엑셀 업로드
- 설명: "지자체로부터 받은 조합원 명단을 업로드합니다"
- 업로드 박스 (드래그앤드롭 UI)
- 샘플 테이블: 성명 | 전화번호 | 주소 | 전입일 (열 4개)
- "양식 다운로드" 버튼
- 업로드 후 미리보기 테이블 (더미 5행: 이○희, 박○수 등)
- 유효성 오류 표시 예시 (전화번호 형식 오류 1건 — 빨간 행)
- "다음: 발송 확인" 버튼

Step 2 · 초대 링크 · 인증코드 발송
- 요약 카드: 총 47명 · 오류 1건 제외 · 발송 대상 46명
- 발송 방식: SMS (기본) / 카카오 알림톡 (옵션)
- 미리보기 메시지:
  "[루트에너지] 곡성 군민 에너지 협동조합 가입 초대
   가입 링크: https://app.rootenergy.co.kr/join?code=ABC123
   인증코드: ABC123 (72시간 유효)
   문의: 061-000-0000"
- "46명에게 발송" 버튼 (클릭하면 "발송 완료" 상태로 전환)

Step 3 · 가입 현황 모니터링
- 실시간 현황 테이블: 이름 | 전화번호 | 발송시각 | 가입여부 | 출자금납입
- 더미 데이터: 일부 가입완료(초록), 일부 미완료(회색), 미납(주황)
- 재발송 버튼 (미가입자 대상)
- 상단 집계: 발송 46 / 가입 완료 31 / 출자금 납입 28

Step 4 · 자격 부여 (권리 확정)
- 설명:
  "전입일 기준으로 조합원 거리 등급을 자동 산정합니다.
   행정 API 연동 시 자동 조회, 미연동 시 엑셀로 전입일 일괄 입력."
- 두 가지 방식 선택 탭:
  A. 행정 API 자동 조회 (admin_api_enabled 플래그 ON 필요)
     → "API 조회 실행" 버튼 + 조회 결과 테이블
  B. 엑셀로 전입일 입력
     → 업로드 박스 + 결과 미리보기
- 거리 등급 결과 테이블: 이름 | 전입일 | 거리등급(인접/인근/일반) | 배당가중치
- 더미 데이터: 인접주민(1.2배) 18명 / 인근(1.0배) 9명 / 일반(0.8배) 4명
- "자격 확정 및 ACTIVE 전환" 버튼 (CORP_ADMIN 권한)

[사이드바 메뉴 구성]
기존 네비게이션에서:
- "AI 에이전트" 하위: "자유 문의"만 표시 (햇빛소득 진단 항목 제거)
- "협동조합" 하위에 "조합원 등록" 추가

[상단 고정 배너]
"🔧 개발자 핸드오프 버전 — 햇빛소득 AI 진단 제외 · 협동조합 온보딩 신규 추가"

[기타]
- styles.css, app.js 재사용. 신규 화면은 인라인 스타일로 보완.
- bg-data.js, char-data.js, gokseong-emd.js script 포함
- 브라우저에서 바로 열림 (서버 불필요)
```
